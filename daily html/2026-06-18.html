Vercel 发布开源 AI 智能体框架 Eve：每个智能体就是一个文件目录

Vercel 开源了 Apache-2.0 许可的 AI 智能体框架 Eve，采用文件系统优先设计：每个智能体对应一个磁盘目录，目录结构直接映射模型、指令、工具、技能、连接、子智能体等能力，无需额外注册代码。内置持久执行、沙箱计算、人机审批、MCP/OpenAPI 安全连接、多通道（Slack/Discord/Teams）以及 OpenTelemetry 追踪与评估能力。Vercel 内部已运行上百个智能体，如数据分析工具 d0、销售代理 Lead Agent（年费约 5000 美元、回报 32 倍）和支持智能体 Vertex。

AI PM 视角：Eve 的战略意义不在于“又一个 Agent 框架”，而在于它把 Agent 的抽象层级从“写代码编排”下沉为“文件目录即配置”。这意味着产品经理和开发者可以用版本控制、代码审查、CI/CD 等成熟工程流程来管理智能体，而不是把 prompt 和工具调用藏在某段 Python 脚本里。对团队而言，Agent 的边界、能力、依赖关系第一次变得像微服务目录一样可读、可审计、可复用。 对竞争格局的影响是直接的：它把 LangChain、CrewAI、Databricks Omnigent 等框架进一步推向“必须证明差异化”的境地。Vercel 自带部署平台与前端生态，其内部 ROI 案例（32 倍回报的销售代理）说明这不是 toy project，而是经过生产验证的范式。信号大于噪声：当 Agent 基础设施收敛到“目录 + MCP + 持久化执行”时，产品护城河将从“谁能写 prompt”转向“谁有高质量工具、可复用技能、可靠评估和真实业务数据”。 这与同日发布的 Google ARD 规范、Databricks Omnigent 形成共振：Agent 生态正在从“单点模型调用”走向“标准化、可发现、可协作的 Agent 网络”。

研究型 Builder 视角：Eve 的技术 novelty 在于将 Agent 的元信息（model、instructions、tools、skills、connections、sub-agents）映射为文件系统树，从而把 Agent 配置变成了可 diff、可 merge、可静态分析的对象。持久执行与 OpenTelemetry 追踪让 Agent 链路具备了可观测性，而沙箱 + 人机审批则提供了生产级安全边界。从研究到产品的路径非常清晰：未来可以在此基础上做 Agent 配置的静态检查、自动能力组合、以及基于 trace 的 RLHF/DPO 优化。

可落地的方法论：- 本周行动：克隆 Eve 仓库，把团队内一条高频重复工作流（如周报生成、客户支持初筛、数据查询）封装成一个 Agent 目录，并在本地沙箱中跑通 10 次执行，记录成功/失败 trace。 - 下季度行动：建立团队级 Agent 目录规范，要求每个 Agent 必须包含 `model.json`、`tools/`、`skills/`、`eval/` 和 `README.md`；把 MCP 服务器接入内部 API，并用 OpenTelemetry 设定北极星指标（成功率、延迟、token 成本）。 ---

NVIDIA GEAR 实验室发布 ENPIRE：8 个 Codex 智能体自主控制机器人完成物理实验

NVIDIA GEAR 实验室推出 ENPIRE 系统，首次实现物理世界自主研究。系统让 8 个 Codex 智能体控制 8 台机器人，配备 GPU 和 token 预算，可通宵无人运行。安全方面采用硬运动极限切断和扭矩受限夹爪两层硬件保障，奖励函数通过视觉分类器离线固定并冻结以防止智能体作弊。系统实时监测机器人利用率（MRU）、token 利用率（MTU）和 GPU 利用率，以 Tokens-to-Success 和 Time-to-Success 评估效率，成功完成扎带、整理细针、安装 GPU 等高精度任务，并将开源。

AI PM 视角：ENPIRE 是一个里程碑：它把 LLM Agent 从“给建议”推进到“在真实物理世界里做实验”。8 个 Codex Agent 并行控制 8 台机器人，意味着科学研究的迭代周期可以从“几周一次人工实验”压缩到“一晚多次自主实验”。这对医药、材料、电子组装、质量检测等行业的 R&D 吞吐量是结构性提升，未来可能出现“Robot/Research as a Service”的云实验室商业模式。 对产品经理的关键启发是：物理世界闭环的产品必须把“安全”和“可度量”放在第一位。ENPIRE 用硬件级硬切断 + 扭矩受限夹爪 + 冻结奖励函数三层设计来防止 reward hacking，这比单纯依赖 prompt 护栏更可靠。竞争壁垒也将从“模型大小”转向“实验协议、安全机制、领域专用 reward 函数和硬件集成深度”。 信号与噪声的判断：单个机器人 demo 已不新鲜，但“多智能体 + 真实物理 + 通宵无人 + 可复现 metrics”是新的信号。它与 OpenAI/Molecule.one 的化学优化、Strands Robots SDK 形成趋势共振：AI 正在接管“感知—规划—执行—验证”的完整实验闭环。

研究型 Builder 视角：ENPIRE 的真正 novelty 在于多 Agent 并行控制真实机器人，并通过冻结的视觉分类器奖励函数来避免智能体在物理环境中作弊。其可复现性较强（系统宣布将开源），且提出的 Tokens-to-Success / Time-to-Success 指标为物理 Agent 的性价比评估提供了基准。机制上看，它揭示了 LLM 作为“高层规划器 + 代码生成器”、机器人作为“执行器”、视觉模型作为“离线评分器”的三层架构可行性，为后续自主湿实验室、自主制造产线提供了可迁移模板。

可落地的方法论：- 本周行动：学习 ENPIRE 的安全设计，为任何涉及物理执行或外部系统调用的 Agent 建立“硬件/软件双层安全 + 冻结评分器”检查清单。 - 下季度行动：在组织内识别一个高重复、可自动化的物理或实验任务，设计最小可行闭环：LLM 生成执行计划 → 传感器/视觉验证 → 人类最终 gate → 自动记录结果，并设定 token 预算上限。 ---

谷歌发布 Agentic Resource Discovery（ARD）开放规范

Google 发布 ARD（Agentic Resource Discovery）开放规范，用于在 Web 上发布、发现和验证 AI 工具、技能与智能体。它基于两个原语：组织在自有域名下托管 catalog 描述可用能力，registry 作为搜索引擎索引 catalog 并响应发现请求。ARD 支持加密验证，使客户端在连接端点前确认发布者身份，然后直接通过原生协议调用能力。Google Cloud 的 Gemini Enterprise Agent Platform 通过 Agent Registry 提供企业级支持，包括 URN 命名、出站策略、工具固定和基于 Agent Identity 的信任验证。

AI PM 视角：当 Agent 数量爆炸时，最大的摩擦不是“能不能调用工具”，而是“敢不敢调用一个陌生工具”。ARD 试图解决的就是 Agent 经济的“信任与发现”问题。它把能力目录放在组织自有域名下，用加密验证确认发布者身份，这让 Agent 工具市场第一次有了类似 HTTPS 的信任基础。对企业产品经理而言，这意味着未来采购或接入第三方 Agent 能力时，可以有可审计的身份链、出站策略和工具固定机制。 对竞争格局的影响在于：Google 再次尝试成为 Agent 生态的“搜索与目录层”。如果 ARD 被广泛采纳，Google Cloud 可以像索引网页一样索引 Agent 能力，进而在企业市场形成网络效应。它与 MCP（传输层）不是竞争而是互补：MCP 负责“怎么连”，ARD 负责“找谁、信谁、管什么”。 信号大于噪声：这是一个协议级发布，而非单一产品功能。它与 Eve 的文件目录、Omnigent 的 Agent 团队一起，勾勒出“Agent 即服务、可发现、可验证”的未来基础设施轮廓。

研究型 Builder 视角：ARD 的技术 novelty 在于把 Web 现有的域名所有权验证和加密身份机制迁移到 Agent 能力发现场景，形成去中心化的 catalog + 中心化 registry 的混合架构。其可复现性高：任何开发者现在就可以在域名下托管 `ai-catalog.json` 并接入 registry。机制上，它把“Agent Identity”作为信任根，支持 URN 命名、出站策略和工具固定，为后续研究 Agent 能力审计、权限最小化、自动 tool selection 提供了标准接口。

可落地的方法论：- 本周行动：为一个公开 API 或内部工具编写 `ai-catalog.json`，尝试通过 ARD registry 验证发现流程，并记录身份验证与策略配置的痛点。 - 下季度行动：设计企业级 Agent 能力目录，规定每个对外/对内 Agent 能力必须包含 ARD 风格的身份、权限、输入输出 schema 和版本信息；在调用链中增加“能力白名单 + 版本固定”机制。 ---

OpenAI 财务文件泄露：高增长背后的巨额亏损

泄露文件显示 OpenAI 2025 年营收 130.7 亿美元（2024 年 37 亿），但研发成本达 191.8 亿美元（含向微软支付 105.9 亿），收入成本（推理计算）75 亿，销售营销成本 57.3 亿，运营亏损 209.2 亿；2025 年净亏损约 390 亿，扣除约 300 亿一次性会计费用后约 80 亿。另有文件显示 2026 年第一季度现金消耗达 37 亿美元，超过同期 57 亿美元收入的一半。OpenAI 正筹备上市，已在美国保密递交 IPO 申请，估值最高可达 1 万亿美元，并已关闭 Sora 视频模型以控制成本。

AI PM 视角：这组财务数据是 2026 年 AI 行业最重要的“冷水”：即使拥有 9 亿周活用户、5000 万付费用户和 130 亿美元年收入，OpenAI 仍未跑通单位经济模型。研发和推理成本合计远超收入，说明“模型能力越大、用户越多、亏损越大”的诅咒尚未解除。这对所有 AI 产品经理的直接影响是：免费或低价补贴的前沿模型时代可能快速收尾，未来产品必须在功能、定价和成本之间做更精细的权衡。 对竞争格局的影响：OpenAI 关闭 Sora、削减非核心业务，意味着它会更加聚焦能直接产生收入的核心对话/API 业务。这为 Databricks、xAI、Anthropic 等对手在垂类场景和成本效率上留下窗口期。同时，IPO 压力会迫使 OpenAI 讲出“盈利故事”，可能带来更激进的企业销售、API 提价和功能裁剪。 信号与噪声：泄露数据需要谨慎看待，但与同日“Q1 现金消耗 37 亿美元”的新闻、以及 Sora 被关闭的事实相互印证。它与 Databricks 80% 增长、Grok 4.3 强调“每美元智能度”形成鲜明对比：AI 产业的下半场竞争，将从“谁模型最强”转向“谁能在给定成本下创造最多业务价值”。

研究型 Builder 视角：这件事本身不是算法突破，却是一个强烈的 reproducibility signal：前沿模型的规模定律在收入侧尚未自我维持。机制上，推理计算的边际成本随用户量线性或超线性增长，而训练成本又随模型能力阶梯式上升；在价格竞争激烈的环境下，毛利率被持续压缩。对研究型 Builder 的启示是：模型选择、推理优化、缓存、蒸馏、MoE、长上下文压缩等“成本工程”研究，将直接决定产品能否规模化盈利。

可落地的方法论：- 本周行动：审计你负责 AI 功能的 per-user 推理成本，按任务类型拆分输入/输出 token、重试率、缓存命中率，并与用户付费或广告价值对比，找出亏损或薄利环节。 - 下季度行动：建立“模型组合”策略：高价值/复杂任务走 frontier 模型，常规任务走更小、更便宜或蒸馏后的模型；引入动态路由和成本预算告警，让每次模型调用都有明确的商业阈值。 ---

Anthropic 与 DeepMind CEO 呼吁 G7 组建排除中国的 AI 联盟

Anthropic CEO Dario Amodei 与 Google DeepMind CEO Demis Hassabis 在 G7 闭门会议上呼吁组建美国主导的联盟，为人工智能制定全球规则和标准。Amodei 指出，该联盟应以前沿模型和硬件（包括芯片及其他关键组件）的访问权限为手段，将中国排除在外。相关评论将其描述为高技术新冷战的开端，竞争方将从根本上被剥夺参与权。

AI PM 视角：两位顶尖 AI 实验室 CEO 在 G7 场合发出这一呼吁，标志着 AI 竞争已从技术层面升级到规则与地缘层面。对产品经理而言，这不再是遥远的政治新闻，而是直接影响市场准入、模型可用性、云区域部署、数据跨境和供应链选择的战略变量。如果联盟成型且中国被排除，全球市场可能分裂为两套平行的模型、芯片和工具生态。 对产品竞争的影响：跨国企业客户会加速提出数据驻留、模型主权、合规审计等要求；中国市场的本地化模型和国产芯片方案将获得更强政策推力；依赖单一云厂商或单一模型供应商的产品会面临更高的监管风险。它与同日“中国加紧筹建世界人工智能合作组织”的消息形成对冲：两套治理叙事正在并行建立。 信号大于噪声：具体的联盟落地尚需时日，但“以模型和硬件访问权为杠杆”的思路已经明确。AI 产品经理需要把地缘政治纳入产品路线图，就像今天把 GDPR、网络安全法纳入合规一样。

研究型 Builder 视角：这并非算法 novelty，而是一种“技术准入机制”研究：前沿模型和先进芯片成为国家战略资源，其分发规则将影响全球研究可及性。机制上，闭源模型的 API 可用性、开源权重的下载权限、芯片出口管制共同构成了一张“研究权限地图”。对研究型 Builder 的启示是：选择开放权重 vs 闭源 API、本地部署 vs 云端调用，不仅是技术决策，也是风险对冲策略。

可落地的方法论：- 本周行动：梳理产品依赖的模型、芯片、云厂商和数据链路，标注出任何单一国家/单一供应商的集中点，识别潜在的地缘与合规风险。 - 下季度行动：设计多区域 AI 架构草案，包括区域化模型部署、数据驻留控制、fallback 模型清单和供应商多元化策略，以减轻单一政策变化带来的冲击。 ---

01

Vercel 发布开源 AI 智能体框架 Eve：每个智能体就是一个文件目录

AI PM 视角：Eve 的战略意义不在于“又一个 Agent 框架”，而在于它把 Agent 的抽象层级从“写代码编排”下沉为“文件目录即配置”。这意味着产品经理和开发者可以用版本控制、代码审查、CI/CD 等成熟工程流程来管理智能体，而不是把 prompt 和工具调用藏在某段 Python 脚本里。对团队而言，Agent 的边界、能力、依赖关系第一次变得像微服务目录一样可读、可审计、可复用。

对竞争格局的影响是直接的：它把 LangChain、CrewAI、Databricks Omnigent 等框架进一步推向“必须证明差异化”的境地。Vercel 自带部署平台与前端生态，其内部 ROI 案例（32 倍回报的销售代理）说明这不是 toy project，而是经过生产验证的范式。信号大于噪声：当 Agent 基础设施收敛到“目录 + MCP + 持久化执行”时，产品护城河将从“谁能写 prompt”转向“谁有高质量工具、可复用技能、可靠评估和真实业务数据”。

这与同日发布的 Google ARD 规范、Databricks Omnigent 形成共振：Agent 生态正在从“单点模型调用”走向“标准化、可发现、可协作的 Agent 网络”。

研究型 Builder 视角：Eve 的技术 novelty 在于将 Agent 的元信息（model、instructions、tools、skills、connections、sub-agents）映射为文件系统树，从而把 Agent 配置变成了可 diff、可 merge、可静态分析的对象。持久执行与 OpenTelemetry 追踪让 Agent 链路具备了可观测性，而沙箱 + 人机审批则提供了生产级安全边界。从研究到产品的路径非常清晰：未来可以在此基础上做 Agent 配置的静态检查、自动能力组合、以及基于 trace 的 RLHF/DPO 优化。

方法论
  • 本周行动：克隆 Eve 仓库，把团队内一条高频重复工作流（如周报生成、客户支持初筛、数据查询）封装成一个 Agent 目录，并在本地沙箱中跑通 10 次执行，记录成功/失败 trace。
  • 下季度行动：建立团队级 Agent 目录规范，要求每个 Agent 必须包含 `model.json`、`tools/`、`skills/`、`eval/` 和 `README.md`；把 MCP 服务器接入内部 API，并用 OpenTelemetry 设定北极星指标（成功率、延迟、token 成本）。

NVIDIA GEAR 实验室发布 ENPIRE：8 个 Codex 智能体自主控制机器人完成物理实验

AI PM 视角：ENPIRE 是一个里程碑：它把 LLM Agent 从“给建议”推进到“在真实物理世界里做实验”。8 个 Codex Agent 并行控制 8 台机器人，意味着科学研究的迭代周期可以从“几周一次人工实验”压缩到“一晚多次自主实验”。这对医药、材料、电子组装、质量检测等行业的 R&D 吞吐量是结构性提升，未来可能出现“Robot/Research as a Service”的云实验室商业模式。

对产品经理的关键启发是：物理世界闭环的产品必须把“安全”和“可度量”放在第一位。ENPIRE 用硬件级硬切断 + 扭矩受限夹爪 + 冻结奖励函数三层设计来防止 reward hacking，这比单纯依赖 prompt 护栏更可靠。竞争壁垒也将从“模型大小”转向“实验协议、安全机制、领域专用 reward 函数和硬件集成深度”。

信号与噪声的判断：单个机器人 demo 已不新鲜，但“多智能体 + 真实物理 + 通宵无人 + 可复现 metrics”是新的信号。它与 OpenAI/Molecule.one 的化学优化、Strands Robots SDK 形成趋势共振：AI 正在接管“感知—规划—执行—验证”的完整实验闭环。

研究型 Builder 视角：ENPIRE 的真正 novelty 在于多 Agent 并行控制真实机器人，并通过冻结的视觉分类器奖励函数来避免智能体在物理环境中作弊。其可复现性较强（系统宣布将开源），且提出的 Tokens-to-Success / Time-to-Success 指标为物理 Agent 的性价比评估提供了基准。机制上看，它揭示了 LLM 作为“高层规划器 + 代码生成器”、机器人作为“执行器”、视觉模型作为“离线评分器”的三层架构可行性，为后续自主湿实验室、自主制造产线提供了可迁移模板。

方法论
  • 本周行动：学习 ENPIRE 的安全设计，为任何涉及物理执行或外部系统调用的 Agent 建立“硬件/软件双层安全 + 冻结评分器”检查清单。
  • 下季度行动：在组织内识别一个高重复、可自动化的物理或实验任务，设计最小可行闭环：LLM 生成执行计划 → 传感器/视觉验证 → 人类最终 gate → 自动记录结果，并设定 token 预算上限。

谷歌发布 Agentic Resource Discovery（ARD）开放规范

AI PM 视角：当 Agent 数量爆炸时，最大的摩擦不是“能不能调用工具”，而是“敢不敢调用一个陌生工具”。ARD 试图解决的就是 Agent 经济的“信任与发现”问题。它把能力目录放在组织自有域名下，用加密验证确认发布者身份，这让 Agent 工具市场第一次有了类似 HTTPS 的信任基础。对企业产品经理而言，这意味着未来采购或接入第三方 Agent 能力时，可以有可审计的身份链、出站策略和工具固定机制。

对竞争格局的影响在于：Google 再次尝试成为 Agent 生态的“搜索与目录层”。如果 ARD 被广泛采纳，Google Cloud 可以像索引网页一样索引 Agent 能力，进而在企业市场形成网络效应。它与 MCP（传输层）不是竞争而是互补：MCP 负责“怎么连”，ARD 负责“找谁、信谁、管什么”。

信号大于噪声：这是一个协议级发布，而非单一产品功能。它与 Eve 的文件目录、Omnigent 的 Agent 团队一起，勾勒出“Agent 即服务、可发现、可验证”的未来基础设施轮廓。

研究型 Builder 视角：ARD 的技术 novelty 在于把 Web 现有的域名所有权验证和加密身份机制迁移到 Agent 能力发现场景，形成去中心化的 catalog + 中心化 registry 的混合架构。其可复现性高：任何开发者现在就可以在域名下托管 `ai-catalog.json` 并接入 registry。机制上，它把“Agent Identity”作为信任根，支持 URN 命名、出站策略和工具固定，为后续研究 Agent 能力审计、权限最小化、自动 tool selection 提供了标准接口。

方法论
  • 本周行动：为一个公开 API 或内部工具编写 `ai-catalog.json`，尝试通过 ARD registry 验证发现流程，并记录身份验证与策略配置的痛点。
  • 下季度行动：设计企业级 Agent 能力目录，规定每个对外/对内 Agent 能力必须包含 ARD 风格的身份、权限、输入输出 schema 和版本信息；在调用链中增加“能力白名单 + 版本固定”机制。

研究型 Builder 视角：这件事本身不是算法突破，却是一个强烈的 reproducibility signal：前沿模型的规模定律在收入侧尚未自我维持。机制上，推理计算的边际成本随用户量线性或超线性增长，而训练成本又随模型能力阶梯式上升；在价格竞争激烈的环境下，毛利率被持续压缩。对研究型 Builder 的启示是：模型选择、推理优化、缓存、蒸馏、MoE、长上下文压缩等“成本工程”研究，将直接决定产品能否规模化盈利。

  • 下季度行动：建立“模型组合”策略：高价值/复杂任务走 frontier 模型，常规任务走更小、更便宜或蒸馏后的模型；引入动态路由和成本预算告警，让每次模型调用都有明确的商业阈值。

MolmoMotion：语言引导的 3D 运动预测模型

分析：AllenAI 发布 MolmoMotion，基于 Molmo 2 骨干网络，输入视频帧、物体上的 3D 点标记及文字动作指令，预测未来数秒内这些点的 3D 轨迹；提供自回归 MolmoMotion-AR 和流匹配 MolmoMotion-FM 两个变体，并开源 MolmoMotion-1M 数据集与 PointMotionBench 基准。

研究型 Builder 视角：技术 novelty 在于 3D 点轨迹预测与流匹配多模态未来运动，配套的大规模标注数据集 PointMotionBench 让研究可复现性显著提升。

方法论

挑选一个需要“语言→空间动作”的产品场景，用 MolmoMotion 替代传统的边界框或关键帧标注，验证端到端交互体验。

3 CROSS-CUTTING MODELS
01

框架 1：Agent 即文件系统——从“编排代码”到“基础设施目录”

今天 Vercel Eve 用文件目录定义 Agent，Google ARD 用域名目录发现 Agent，Databricks Omnigent 用团队方式组合 Agent。三者的共同趋势是：Agent 正从“写代码调 API”的临时脚本，变成可版本化、可发现、可组合的基础设施对象。

  • 因果链：目录化抽象降低心智负担 → 更多开发者能定义 Agent → Agent 数量激增 → 发现与信任问题凸显 → ARD 等目录/身份协议成为刚需。
  • 结构洞察：下一代 Agent 平台很可能呈现“文件系统定义 + MCP 工具连接 + ARD 能力发现”的三层架构。

产品启示：产品经理应尽早把 Agent 视为“可复用组件”而非“一次性功能”。在规划阶段就定义每个 Agent 的输入输出 schema、成功指标、工具依赖和失败回退；在团队层面建立 Agent 目录、版本管理和访问控制。不要先写复杂编排器，而是先让 Agent 的边界和契约清晰。

02

框架 2：物理世界闭环——从“数字建议”到“自主实验”

ENPIRE 让多 Agent 控制真实机器人完成物理实验，Strands SDK 打通 Hugging Face Hub 到机器人硬件，MolmoMotion 用语言预测 3D 点轨迹，OpenAI/Molecule.one 用 GPT-5.4 优化化学反应。这些事件共同指向一个闭环：语言指令 → 空间/动作预测 → 物理执行 → 传感器验证 → 奖励信号。

  • 因果链：语言模型提供高层语义理解 → 3D/动作模型弥合语义到物理的鸿沟 → 机器人/实验设备执行 → 离线冻结的评分器保证可复现性 → 多 Agent 并行加速探索。
  • 结构洞察：物理 Agent 的可靠性不取决于模型参数大小，而取决于“安全切断 + 冻结评分 + 可观测 metrics”的工程闭环。

产品启示：如果你的产品涉及现实世界操作（制造、物流、实验、医疗），不要急于让 LLM 直接控制硬件。先定义清晰的“物理 API”、传感器验证方案和不可绕过的安全层；再把 LLM 放在“规划与代码生成”层，而不是“最终执行决定”层。产品价值来自闭环的可靠性，而不是 demo 的惊艳。

03

框架 3：成本-地缘双约束下的 AI 可持续性

OpenAI 的巨额亏损与现金消耗、G7 技术联盟、中国筹建世界 AI 合作组织、Databricks 的高增长、Sora 被关闭，共同描绘了一个新背景：前沿 AI 的增长不再只受技术驱动，而是受单位经济模型和地缘政治双重约束。

  • 因果链：推理与训练成本增速超过收入 → 厂商被迫提价、裁剪功能或聚焦高价值场景 → 地缘分裂增加合规与供应链成本 → 企业客户更重视成本效率与部署可控性 → 数据+AI 一体化平台和高效模型获得优势。
  • 结构洞察：未来的产品竞争力 = （业务价值 / token 成本） × （合规可控性）。

产品启示：AI 产品策略必须假设“模型不会无限变便宜、政策不会无限开放”。优先做三件事：一是建立按任务价值分层的模型调用策略；二是把数据和模型推理尽量放在同一可控区域；三是为每个 AI 功能建立明确的单位经济模型。不要再以“先补贴用户、再想办法盈利”作为默认假设。

Thinking ///

5 DEEP QUESTIONS
01

思考题 1：当 Agent 能力可以通过文件目录或 ARD 目录被任意组合时，产品的护城河将从“功能数量”转向什么？

Vercel Eve 和 Google ARD 让 Agent 的定义、发现、调用变得标准化。这意味着“有没有某个 Agent”将不再稀缺，稀缺的是如何让 Agent 在真实业务中可靠地产生价值。

  • [产品战略]：护城河应转向“领域数据飞轮 + 专属评估体系 + 深度工作流集成”。
  • [执行/团队]：团队需要平台工程师、领域专家和 eval 工程师的三角协作，而不是单纯堆积 prompt 工程师。
  • [风险/机会]：基础 Agent 能力会快速商品化，但“知道在哪些场景、以什么约束、用什么指标组合 Agent”将成为新的差异化高地。
02

思考题 2：自主物理实验系统（ENPIRE）的“奖励函数冻结”与“硬件安全切断”能否被迁移到高风险业务场景？

ENPIRE 通过冻结视觉分类器和硬件级运动极限来防止 Agent 在物理世界中作弊或越界。医疗、金融、制造等领域同样需要类似的“不可绕过”的安全层。

  • [产品战略]：任何让 Agent 直接作用于真实世界的功能，都应先定义失败模式和硬边界，再讨论自动化程度。
  • [执行/团队]：安全工程应成为 AI 产品团队的核心能力，而不是事后由合规部门补票。
  • [风险/机会]：过度谨慎会牺牲效率，但一次严重事故足以摧毁用户信任；关键是把安全设计为可审计、可回滚、可解释的机制。
03

思考题 3：OpenAI 的高亏损是否预示着“模型能力免费/低价补贴”时代的结束？

OpenAI 即使拥有 9 亿周活用户和 130 亿美元收入，仍未跑通单位经济模型。这迫使整个行业重新审视“先规模、后盈利”的 AI 增长逻辑。

  • [产品战略]：免费 tier 应被视为获客成本而非默认策略；每个 AI 功能都需要明确的变现路径或成本分摊机制。
  • [执行/团队]：产品经理需要与成本工程、定价团队更紧密协作，把“token 预算”纳入需求文档。
  • [风险/机会]：补贴退坡可能让中小型玩家更难获客，但也为成本效率更高、模型选择更灵活的团队创造窗口期。
04

思考题 4：G7 技术联盟与中国 AI 合作组织并存，是否会导致两套平行的模型/工具/数据生态？

Anthropic 与 DeepMind CEO 呼吁排除中国的 AI 联盟，与中国筹建世界 AI 合作组织形成对冲。全球 AI 治理可能进入“双轨制”。

  • [产品战略]：跨国产品必须支持多区域模型部署、数据驻留和供应商 fallback，而不是绑定单一云或单一模型。
  • [执行/团队]：合规、法务和基础设施团队需要前置参与 AI 产品路线图，避免政策变化导致功能下架。
  • [风险/机会]：生态分裂会增加开发和维护成本，但也会催生本地化和主权 AI 服务的新市场。
05

思考题 5：当 Claude Design、Eve、Omnigent 等工具让非专业用户也能生成代码/应用/Agent，产品经理的价值定位应如何升级？

生成工具的普及会降低“把需求写成代码”的门槛，但不会降低“定义正确问题、约束边界、评估结果”的难度。

  • [产品战略]：产品经理的价值将从“写需求文档”转向“设计目标、约束条件、评估标准和人在环策略”。
  • [执行/团队]：未来团队可能是“人类 PM + 多个 AI Agent”的模式，PM 需要学会分配任务、仲裁冲突和验收 Agent 输出。
  • [风险/机会]：速度会大幅提升，但质量、安全和一致性风险也会放大；谁能定义好“什么算好”，谁就能驾驭 AI 团队。
